# CI Rules for Materials Plane v1
## 1. Objectives

These rules define continuous integration (CI) checks for substrates and materials under Materials Plane v1. They implement the policies:

- "no corridor, no build": all required materials corridors must be present and finite; and
- "corridor_ok = true": only substrates whose material_risks shards pass corridor checks may enter production lanes.

## 2. Inputs

CI operates on three artifact families:

1. Corridor tables: ecosafety.corridors.v2 shards with var_id rows for all required materials coordinates (t90, tox, micro, leach_cec, pfas, caloric, materials).
2. Material kinetics: ecosafety.material_kinetics.v1 shards describing candidate substrates.
3. Material risks: ecosafety.material_risks.v1 shards generated by trusted kernels from the kinetics plus corridor tables.

## 3. Rule: No Corridor, No Build

For every ecosafety.material_kinetics.v1 particle with material_id = M:

- CI MUST verify that ecosafety.corridors.v2 contains mandatory rows for:
  - MAT.T90
  - MAT.TOX
  - MAT.MICRO
  - MAT.LEACH_CEC
  - MAT.PFAS
  - MAT.CALORIC
  - MAT.MATERIALS

If any mandatory row is missing, or any of its safe/gold/hard fields is NaN or otherwise invalid:

- CI MUST fail the pipeline.
- No crate or manifest referencing material_id = M MAY be marked production-ready.

This ensures that all materials risk coordinates are defined before any substrate is considered deployable.

## 4. Rule: Corridor_ok = True for Production

For every ecosafety.material_risks.v1 particle with material_id = M:

- CI MUST validate that:
  - 0 ≤ r_t90, r_tox, r_micro, r_leach_cec, r_pfas, r_caloric, r_materials ≤ 1;
  - corridor_ok == true for production lanes.

Production manifest rules:

- If a deployment manifest, node configuration, or substrate registry entry marks material_id = M as lane = "PROD" or "EXPPROD", then:
  - CI MUST confirm at least one ecosafety.material_risks.v1 shard for M with corridor_ok == true and spec_id/spec_hash matching the manifest.
  - If no such shard exists, or corridor_ok == false, CI MUST fail and mark the configuration as non-deployable.

Research lanes:

- Materials with corridor_ok == false MAY appear only in lane = "RESEARCH" configurations.
- CI MUST ensure that research-only manifests do not link to actuation-enabled nodes in production environments (e.g., by requiring lane = "RESEARCH" also in node manifests).

## 5. Rule: Consistency with Spine RiskVector

For any qpudatashard.ecosafety.v1 row where material_id = M is non-empty:

- CI MUST check that:
  - There exists a matching ecosafety.material_risks.v1 shard for M.
  - r_materials in qpudatashard.ecosafety.v1 equals r_materials in the material_risks shard within a small tolerance.
  - If the node lane is "PROD" or "EXPPROD", then corridor_ok in the associated material_risks shard is true.

If any of these conditions are violated, CI MUST fail and the offending shard MUST NOT be accepted into the long-term evidence ledger.

## 6. Rule: KerDeployable and Materials Plane

In addition to spine-level KER thresholds, CI MUST enforce:

- kerdeployable == false for any configuration that references:
  - a material_id with corridor_ok == false; or
  - a material_id whose r_materials ≥ materials hard band; or
  - missing materials corridors.

Only configurations with:

- kerdeployable == true,
- corridor_ok == true for all referenced materials, and
- complete materials corridor coverage

MAY be promoted to production routing and actuation workflows.

## 7. Implementation Notes

These rules are to be implemented as:

- Rust-based CI checks that:
  - parse ecosafety.corridors.v2, ecosafety.material_kinetics.v1, ecosafety.material_risks.v1, qpudatashard.ecosafety.v1;
  - compute required joins by material_id and spec_hash; and
  - return non-zero exit status on any violation.

- Repository-level policies that:
  - prevent merging changes that introduce new substrates without corresponding material_kinetics and material_risks shards;
  - prevent enabling new materials in production manifests until all checks above pass.

By enforcing these rules, the materials plane remains a first-class, Lyapunov-consistent risk coordinate and unsafe substrates are structurally blocked from deployment.
